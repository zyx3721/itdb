# 旧平台数据库导入迁移功能说明

## 📋 功能概述

本功能实现了从旧平台数据库到新平台数据库的自动迁移转换，主要处理 `settings` 和 `statustypes` 两个表的结构差异。

## 🔄 迁移内容

### 1. settings 表迁移

**旧结构**（12 个字段）：
```
companytitle, dateformat, currency, lang, version, timezone, dbversion,
useldap, ldap_server, ldap_dn, ldap_getusers, ldap_getusers_filter
```

**新结构**（7 个字段）：
```
useldap, ldap_server, ldap_dn, ldap_bind_dn, ldap_bind_password,
ldap_getusers, ldap_getusers_filter
```

**迁移逻辑**：
- 保留所有 LDAP 相关字段
- 删除旧平台特有的字段（companytitle, dateformat, currency, lang, version, timezone, dbversion）
- 新增 ldap_bind_dn 和 ldap_bind_password 字段（默认为空）

### 2. statustypes 表迁移

**旧结构**（2 个字段）：
```
id, statusdesc
```

**新结构**（3 个字段）：
```
id, statusdesc, color
```

**迁移逻辑**：
- 添加 `color` 字段（TEXT 类型）
- 为固定的 4 个状态类型设置颜色：
  - **使用中**: `#2f7fba` （蓝色）
  - **库存**: `#16a34a` （绿色）
  - **有故障**: `#dc2626` （红色）
  - **报废**: `#9ca3af` （灰色）
- 其他自定义状态类型的颜色为空

## 🚀 使用方式

1. 启动后端服务
2. 登录系统
3. 进入"系统设置" → "数据库管理" → "导入数据库"
4. 选择旧平台数据库文件（itdb.db）
5. 点击"上传导入"
6. 系统自动执行迁移转换
7. 导入成功后，数据库结构已自动适配新平台

## 🔒 安全机制

### 1. 备份保护
- 导入前自动备份当前数据库到 `data/backups/` 目录
- 备份文件命名格式：`import-database_YYYYMMDD_HHMMSS.db`
- 迁移失败时自动回滚到备份

### 2. 事务保护
- 所有迁移操作在事务中执行
- 任何步骤失败都会回滚，保证数据一致性

### 3. 验证机制
- 导入前验证数据库完整性（PRAGMA integrity_check）
- 验证是否包含必需的 items 表
- 迁移后验证表结构是否正确

### 4. 锁机制
- 导入期间加锁，防止并发访问
- 确保数据库替换过程的原子性

## 📊 迁移流程图

```
上传旧数据库
    ↓
验证数据库完整性
    ↓
备份当前数据库
    ↓
关闭当前数据库连接
    ↓
替换数据库文件
    ↓
打开新数据库
    ↓
执行迁移转换 ← 【核心功能】
    ├─ 迁移 settings 表
    └─ 迁移 statustypes 表
    ↓
验证迁移结果
    ↓
创建默认 admin 用户（如果没有用户）
    ↓
导入成功
```

## 🛠️ 技术实现

### 核心函数

#### 1. `migrateOldDatabaseSchema(db *sql.DB) error`
主入口函数，协调整个迁移流程。

#### 2. `migrateSettingsTable(db *sql.DB) error`
迁移 settings 表：
- 检测旧结构（是否包含 companytitle 等字段）
- 创建新结构表 `settings_new`
- 复制 LDAP 相关字段数据
- 删除旧表，重命名新表

#### 3. `migrateStatusTypesTable(db *sql.DB) error`
迁移 statustypes 表：
- 检测是否缺少 color 字段
- 创建新结构表 `statustypes_new`
- 复制数据并为固定状态类型设置颜色
- 删除旧表，重命名新表

#### 4. `updateStatusTypesColors(db *sql.DB) error`
更新已有 color 字段的表：
- 为固定状态类型补充颜色（如果为空）

#### 5. `getTableColumns(db *sql.DB, tableName string) ([]string, error)`
工具函数，获取表的所有列名。

## 📝 注意事项

### 1. 兼容性
- 自动检测数据库结构，只对旧结构执行迁移
- 如果已是新结构，跳过迁移
- 支持重复导入（幂等性）

### 2. 数据保留
- settings 表：保留所有 LDAP 配置
- statustypes 表：保留所有状态类型，只添加颜色字段
- 其他表：完全保留，不做任何修改

### 3. 性能
- 迁移过程在事务中执行，速度快
- 测试数据库（348KB）迁移耗时约 30ms

### 4. 日志
- 迁移过程输出详细日志
- 便于排查问题和审计

## 🎯 适用场景

1. **从旧平台迁移到新平台**
   - 旧平台数据库包含完整业务数据
   - 需要保留所有数据并适配新平台结构

2. **数据库结构升级**
   - settings 表结构简化
   - statustypes 表增加颜色支持

3. **批量导入测试**
   - 支持多次导入测试
   - 自动备份，安全可靠
