## 📋 版本概述

v1.0.0 是 itdb 的首个正式版本，基于原 PHP + SQLite 项目重构为 Go + Vue3 前后端分离架构。该版本覆盖 IT 资产、软件许可、合同、发票、文件、机架、地点、字典、用户和系统设置等核心管理场景，并提供 Docker Compose 部署、自动备份与保留期限清理、数据库导入迁移、LDAP 登录、操作审计、CI/CD 构建发布和 Swagger/OpenAPI 文档能力。

---

## ✨ 新增特性

- 新增 IT 资产全生命周期管理，覆盖硬件、软件、合同、发票、文件、厂商和代理商
- 新增地点、区域、机架和机架可视化能力
- 新增本地账号与 LDAP 双模式登录
- 新增管理员完全访问和普通用户只读权限
- 新增操作历史、浏览历史和 Excel 导出
- 新增标签打印、二维码生成和多种标签纸预设
- 新增旧版 SQLite 数据库导入与自动迁移能力
- 新增每日 SQLite 自动备份和 Schema 变更前备份
- 新增 `ITDB_DAILY_BACKUP_RETENTION_DAYS` 环境变量，可配置每日自动备份保留天数
- 新增 Docker Compose、Nginx、Supervisor 和容器入口部署配置
- 新增后端 Air 本地热重载配置，便于开发环境调试
- 新增 Swagger UI 与 OpenAPI JSON/YAML 文档，覆盖后端控制中心接口
- 新增 README 项目截图、技术栈、项目结构、部署和 API 文档说明

## 🚀 性能优化

- 使用纯 Go SQLite 驱动，降低部署环境对 CGO 和系统库的依赖
- 通过 Vite 构建前端静态资源，提升开发和部署效率
- Docker 镜像采用多阶段构建，减少运行镜像体积
- 每日自动备份支持按保留天数清理历史文件，降低长期运行时磁盘占用

- 规范前端 Prettier 格式化配置和 TypeScript 工程配置
- 扩展 `.gitignore` 与 `.dockerignore`，排除运行时数据、构建产物和本地环境文件

## 🔄 升级说明

这是首个正式版本，无历史版本升级要求。

部署前请注意：

1. 生产环境务必修改默认管理员密码 `admin / admin123`
2. 生产环境务必设置 `ITDB_JWT_SECRET`
3. 使用 Docker Compose 部署时，请参考 `deploy/.env.example` 创建 `deploy/.env`
4. 从旧版 PHP ITDB 迁移时，可通过系统内置数据库导入功能上传旧版 `.db` 文件
5. 如需自动清理每日备份，可设置 `ITDB_DAILY_BACKUP_RETENTION_DAYS` 为正整数
6. 请定期备份 `backend/data/` 或 Docker 持久化数据目录
